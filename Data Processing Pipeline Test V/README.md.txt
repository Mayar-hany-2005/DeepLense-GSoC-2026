# Data Processing Pipeline Test V Documentation

This folder contains the data processing and binary classification pipeline for LSST mock data (Lenses vs. Non-Lenses), specifically designed to handle highly imbalanced datasets.

## Files

* **`Data_Processing_Pipeline_for_the_LSST.ipynb`**
  * Jupyter notebook implementing the lens finding pipeline.
  * Explains the strategy for overcoming severe class imbalance using PyTorch's `WeightedRandomSampler` and on-the-fly geometric data augmentations.
  * Produces ROC curves, AUC scores, and training validation curves for model comparison (ResNet18 vs EfficientNet-B0).
  * **To use:** Open in Jupyter/Colab, ensure the datasets are loaded, and run all cells.

* **`best_model_efficientnet_b0.pth` & `best_model_resnet18.pth`**
  * Trained PyTorch model weights for the pipeline.
  * Generated by the training process described in the notebook.
  * *(Note: Due to GitHub file size limits, these might be hosted via an external Google Drive link provided in the notebook/main repo).*
  * **To use:** Loaded automatically by the notebook at the evaluation stage to reproduce the final metrics.

## Outputs
* ROC curves, AUC scores, and training metrics are generated dynamically by running the notebook.
* Outputs are displayed inline and can be exported as `.png` files.

## Reviewer Notes
* The dataset is highly imbalanced (Non-lenses >>> Lenses); the implementation relies on weighted sampling rather than raw accuracy. Please refer to the AUC score for true performance evaluation.
* Follow the dataset loading instructions in the notebook's opening cells before running the pipeline.
